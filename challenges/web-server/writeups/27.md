**So the `index?page=` is programmed nicely that %00 null byte doesn't removes the .php form `?index?page=./admin.html%00.php`.**
<br><br>
**After researching a bit on truncation in php as the challange name suggests I found out that if the request is greater than 4096 bytes then it truncates/removes it works till php version 5.x.x so I viewed page technologies using wappalyzer and the php version was same as our requirement.**

**When researching based on the path truncation documentation it was found that we cannot attack on it by giving the absolute path but can which a path that is not real so we will use `x/../admin.html` which resolves to `admin.html` and its not a actual path as required. And we will use `././././` as padding at end because it resolves to same current dir .Now we will craft a bash payload:**
<br><br>
```bash
#!/usr/bin/env bash
BASE_URL="http://challenge01.root-me.org/web-serveur/ch35/index.php?page=x/../admin.html/"
PAYLOAD=$(printf './%.0s' {1..2048})  # ./ = 2 bytes and ./ × 4096 = 8192 bytes so 2048 times
curl "${BASE_URL}${PAYLOAD}"
```
